Net Working and the Internet
TERM DEFINITION
Network Classifications
  • LAN: Local Area Network
  • MAN: Metropolitan Area Network
  • WAN: Wide Area Network
    • Network Models
  • Bus Network: A network topology where all devices share a single communication line, allowing data to be transmitted to all devices, but only the intended recipient processes the information.
  • Star Network: A network topology where all devices are connected to a central hub or switch, with data flowing directly between the central hub and individual devices.
    		•
    Protocols
  • CSMA/CD Ethernet: Carrier Sense Multiple Access with Collision Detection, an Ethernet protocol that regulates access to a shared communication medium, allowing devices to detect and handle collisions.
  • CSMA/CA WiFi: Carrier Sense Multiple Access with Collision Avoidance, a protocol used in wireless networks (WiFi) to manage access to the shared channel by avoiding collisions through careful scheduling and coordination.
    		•
    Combining Networks
  • Repeater: A network device that amplifies and retransmits signals to extend the range of a network, facilitating communication between distant nodes.
  • Bridge: A device that connects and filters traffic between different segments of a network, operating at the data link layer and enhancing network performance and security.
  • Switch: A network device that operates at the data link layer, efficiently forwarding data only to the specific device it is intended for within a local area network (LAN).
  • Router: A device that connects different networks, directing data packets between them based on their destination IP addresses, facilitating communication across multiple networks.
  • Gateway: A network node that connects different types of networks, translating protocols and enabling communication between networks with different architectures or communication protocols.
    		•
    Client/ Server Model
  • P2P (Peer-to-peer): A decentralized network model where participants (peers) share resources directly with each other, rather than relying on a centralized server.
  • Distributed System: A system architecture where tasks or processes are distributed across multiple computers, working together to achieve a common goal or provide a service.
  • Cloud Computing: A computing paradigm that involves delivering various services over the internet, allowing users to access and use computing resources (such as servers, storage, databases) on-demand.
    		•
    Internet Addressing
  • Tier 1: The highest level in the hierarchical structure of internet service providers, consisting of major global networks that exchange traffic directly without intermediaries.
  • Tier 2: Regional or national internet service providers that connect to Tier 1 providers and serve as intermediaries, providing internet access to a specific geographic area.
  • Tier 3: Local internet service providers that connect to Tier 2 providers, serving end-users and businesses in specific localities with direct internet access.
    		•
    IP Address
  • ICANN: ( https://www.icann.org/ ) Internet Corporation for Assigned Names and Numbers, a nonprofit organization responsible for coordinating the maintenance and procedures of several databases related to the namespaces and numerical spaces of the Internet.
  • DNS: Domain Name System, a hierarchical system that translates human-readable domain names into IP addresses, facilitating the navigation of the internet.
  • TLD: Top-Level Domain, the highest level in the hierarchical domain name system, such as .com, .org, and .net.
  • Country-code TLDs: ( https://www.worldstandards.eu/others/tlds/ ) Top-Level Domains assigned to specific countries or territories, providing a way to associate a website with a particular geographic location, for example, .us for the United States or .uk for the United Kingdom.
    		•
    Internet Protocols
  • FTP: (File Transfer Protocol) A standard network protocol used to transfer files from one host to another over a TCP-based network, such as the Internet.
  • HTTP: (Hypertext Transfer Protocol) The foundation of any data exchange on the Web, enabling the transfer of hypertext or web pages between clients and servers.
  • SMTP: (Simple Mail Transfer Protocol) A standard protocol for sending and receiving electronic mail, allowing the transmission of email messages between servers.
    		•
    World Wide Web
  • URL: (Uniform Resource Locator) A web address that specifies the location of a resource on the Internet, typically starting with "http://" or "https://".
  • Client-side activities: Actions or processes that occur on the user's device within a web browser, such as rendering web pages, executing scripts, and handling user interactions.
  • Server-side activities: Actions or processes that occur on the web server in response to client requests, including processing data, handling database queries, and generating dynamic content before sending it to the client.
    		•
    Internet four-layer software
  • Application layer: The top layer of the Internet protocol suite responsible for providing network services directly to end-users or applications, including protocols for email, file transfer, and other high-level functions.
  • Transport layer: The layer that ensures end-to-end communication and data flow control, responsible for error detection, correction, and data segmentation into smaller units (segments).
  • Network Layer: The layer responsible for routing and forwarding data packets between devices on different networks, enabling communication across multiple networks and addressing through logical addressing.
    		•
    Internet is
  • Packet-switching network: A type of network where data is broken into packets for efficient transmission, with each packet taking its own route to the destination and being reassembled upon arrival.
  • Fault-tolerant: A system or network's ability to continue operating with minimal disruption in the presence of faults or errors, ensuring reliability and continuous functionality.
  • Redundant: Having backup or duplicated components or systems in place to ensure continued operation and reduce the risk of failure in the event of a malfunction or failure in one part of the system.
    		•
    Cybersecurity
  • Malware: Malicious software designed to harm or exploit computer systems, including viruses, worms, trojan horses, spyware, and other harmful programs.
  • Virus: A type of malware that attaches itself to legitimate programs or files and spreads when the infected program or file is executed.
  • Worm: Self-replicating malware that spreads across computer networks without user intervention, often exploiting vulnerabilities in the network.
  • Trojan Horse: Malicious software disguised as legitimate software, tricking users into installing it and allowing unauthorized access or harm to the system.
  • Spyware: Software designed to secretly collect and send user information without their knowledge, often for advertising or malicious purposes.
  • Phishing: A fraudulent attempt to obtain sensitive information, such as usernames, passwords, and credit card details, by disguising as a trustworthy entity in electronic communication.
  • DDoS attack: Distributed Denial of Service attack, where multiple compromised computers are used to flood a target system with traffic, causing a temporary or permanent disruption of services.
    		•
    Protection
  • firewall: A security system designed to monitor and control incoming and outgoing network traffic based on predetermined security rules.
  • spam filter: Software that identifies and filters out unwanted and unsolicited email messages to protect against spam.
  • proxy server: An intermediary server that acts as a gateway between a user's device and the internet, enhancing security and privacy by hiding the user's IP address.
  • antivirus software: Programs designed to detect, prevent, and remove malicious software (malware) from computer systems, safeguarding against viruses and other threats.
  • cryptography: The practice and study of techniques for secure communication in the presence of third parties, involving mathematical algorithms for encryption and decryption.
  • Public Key encryption: A cryptographic system that uses a pair of keys (public and private) to enable secure data transmission and ensure the confidentiality and integrity of digital communication.
